# Weather Forecasting for Critical Conditions

## Краткое описание

Разработка модели машинного обучения для краткосрочного прогнозирования вероятности возникновения критических погодных условий (интенсивные осадки, шквальный ветер) на основе локальных метеорологических измерений с интеграцией глобальных прогнозов и foundation-модели TimesFM.

---

## Основные характеристики

| Характеристика | Значение |
|----------------|----------|
| **Тип задачи** | Бинарная классификация с вероятностным выходом |
| **Входные данные** | Многомерные временные ряды (64 минуты) |
| **Выход** | Вероятность события |
| **Архитектура** | Conv1D + Stacked LSTM + Attention |

---

## Архитектура модели

```
Вход (3 уровня данных)
    ↓
Conv1D (выделение локальных паттернов)
    ↓
Stacked LSTM (долгосрочные зависимости)
    ↓
Attention (фокус на значимые интервалы)
    ↓
Dense + Sigmoid (вероятность события)
```

### Компоненты модели

| Компонент | Параметры | Назначение |
|-----------|-----------|------------|
| Conv1D | kernel=3-5, filters=32-64 | локальные паттерны и аномалии |
| LSTM (слой 1) | 256-512 нейронов | кодирование последовательности |
| LSTM (слой 2) | 128-256 нейронов | агрегация информации |
| Attention | линейный слой + softmax | выделение значимых интервалов |
| Dense | 64-128 нейронов, ReLU | нелинейное преобразование |
| Output | 1 нейрон, Sigmoid | вероятность [0, 1] |

### Регуляризация
- Dropout: 0.1-0.3 (предотвращение переобучения)
- Batch Normalization (ускорение сходимости)

## Источники данных (3 уровня)

### 1. Локальные измерения (метеокомплекс МАМКА)

| Тип сенсора | Измеряемые параметры |
|-------------|---------------------|
| WindDetectorPWS200Z | скорость/направление ветра, вертикальные потоки |
| TempHumidityICB330-01 | температура, влажность, давление |
| SolarRadiation | полная, прямая и рассеянная радиация |
| PrecipitationICB-RS-2E | интенсивность, накопление, тип осадков |
| DustPWD200 | PM10, PM2.5 (концентрация пыли) |
| NoiseDetectorICB30006 | уровень акустического шума |

**Формат хранения:** HDF5 в S3-совместимом хранилище

### 2. Глобальные прогнозы (MOS-подход)

| Источник | Параметры | Частота |
|----------|-----------|---------|
| Open-Meteo API | температура, ветер, давление, осадки | часовая → интерполяция до 1 мин |

### 3. Эмбеддинги foundation-модели

| Модель | Характеристики | Назначение |
|--------|----------------|------------|
| TimesFM_2p5_200M_torch | 200M параметров, обучена на 100B+ точек | компенсация недостатка локальных данных |

## Предобработка данных

| Этап | Метод | Назначение |
|------|-------|------------|
| Ресемплирование | 1 минута | синхронизация данных |
| Обработка пропусков | линейная интерполяция + forward fill | восстановление пропущенных значений |
| Фильтрация | удаление выбросов (NaN, ±∞) | очистка от аномалий |
| Нормализация | Min-Max scaling | приведение к единому диапазону |
| Формирование признаков | лаги (1,3), скользящее среднее (5), разности | учёт динамики |
| Скользящее окно | длина 64 шага | формирование обучающих выборок |

---

## Метрики оценки

| Метрика | Назначение | Целевое значение |
|---------|------------|------------------|
| **AUC-ROC** | способность различать классы (главная метрика) | > 0.8 |
| **MAE** | средняя абсолютная ошибка | < 0.1 |
| **MSE** | штраф за крупные ошибки | чем меньше, тем лучше |
| **R²** | доля объяснённой дисперсии | ближе к 1 |
| **Baseline improvement** | улучшение над тривиальной моделью | > 20% |

## Структура проекта

```
├── MamkaLoader          # загрузка данных из S3 (HDF5)
├── WeatherPipeline      # предобработка и создание признаков
├── GlobalWeatherLoader  # загрузка глобальных прогнозов (Open-Meteo)
├── TimesFMExtractor     # извлечение эмбеддингов TimesFM
├── AttentionFusionLSTM  # основная модель (LSTM + Attention)
├── batch_generator      # динамическая генерация батчей
├── train_model          # обучение модели
└── evaluate_model       # оценка и визуализация
```
